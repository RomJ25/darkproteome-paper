# manuscript/ — what is canonical, what is not

- **`manuscript.md`** — the **CANONICAL paper**. Everything else here is derived from it.
- **`figures/`** — Figs 1–4 (PDF + PNG), generated by `make_figures.py`.

**Build artifacts — do NOT hand-edit (regenerate instead):**
- `manuscript.html` and `manuscript.pdf` are generated from `manuscript.md`. `manuscript.pdf` is
  tracked (committed) so it renders inline in GitHub's file viewer, no `tectonic` install needed.
  Its bytes aren't reproducible run-to-run (tectonic embeds a build timestamp) -- its *content*
  is what's guaranteed to match `manuscript.md`. After editing `manuscript.md`, regenerate both
  (`python3 manuscript/md_to_html.py`, then the tectonic build below), then commit the refreshed
  `manuscript.pdf`.
- `tex/` — the LaTeX (tectonic) typeset build of the paper: `manuscript.tex` (wrapper, also
  defines the math macros) + `_body.tex` (the `manuscript.pdf` / `_preview` / aux outputs in
  this directory are gitignored -- an intermediate, not the copy readers see). Rebuild with
  `tectonic manuscript/tex/manuscript.tex --outdir manuscript/tex`, then copy the resulting PDF
  to `manuscript/manuscript.pdf`. Uses the XeTeX-native `libertinus` package (Libertinus Serif +
  unicode-math).
